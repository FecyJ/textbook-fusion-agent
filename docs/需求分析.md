# 需求分析

## 子问题分解

本项目围绕“多本教材压缩到不超过 30% 精华版本并保持教学效果”拆成六个子问题：

1. 教材解析：把 PDF、Markdown、TXT、DOCX 转成统一的教材-章节结构。
2. 知识点粒度定义：以章节内可独立讲授、可被问答引用的概念、机制、现象、方法作为知识点。
3. 重复判定：先按知识点名称规范化和字符 ngram 相似度找候选，再用 LLM 输出或系统规则给出 merge/keep/remove 决策。
4. 教学连贯性保障：保留唯一知识点，合并重复知识点时保留较完整定义，并保留图谱关系和 RAG 原文引用。
5. 压缩比计算：`整合后知识点定义总字数 / 原始教材正文总字数`，系统在整合统计中展示。
6. RAG 问答：所有回答都基于检索 chunk，并返回教材、章节、页码和原文片段。

## P0 功能边界

- 多格式加载：首版支持 PDF、Markdown、TXT、DOCX；Excel 不作为 P0 阻塞。
- PDF 解析：使用 PyMuPDF 逐页提取文本，按“第X章/Chapter X”识别章节；无法识别时按页段兜底。
- 知识图谱：优先调用配置的大模型服务抽取 JSON；失败时用关键词启发式兜底，保证演示闭环不中断。
- 跨教材整合：首版用 TF-IDF 字符 ngram 相似度分组，再生成解释性决策。
- RAG：首版用 TF-IDF + BM25 混合检索，后续可替换 embedding 模型。
- 教师反馈：首版支持自然语言覆盖匹配到的整合决策，并保留会话历史。

## 分块策略

RAG 按章节正文切成约 700 字 chunk，相邻 chunk 重叠 80 字。这个粒度兼顾三点：

- 单个 chunk 足够包含课程概念定义和上下文。
- top-5 注入 LLM 时不会过长。
- 80 字重叠降低定义跨 chunk 被切断的概率。

## 风险

- 大体量 PDF 完整解析和全章 LLM 抽取耗时较高。
- 首版图谱和整合算法偏工程可用，不等同于人工教研级别精度。
- 没有真实 embedding 模型时，跨语言同义词需要依赖 LLM 或别名字典补强。
