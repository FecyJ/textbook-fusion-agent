# 创新点说明

本页面面向评分标准 F“创新与自由发挥”。每个创新点均给出实现位置、设计理由和可验证效果，避免只描述功能清单。

F 维度容易被 A-E 进阶项吸收，因此这里把创新点限定为“超出基础 P0/P1 的工程闭环、可解释能力或体验整合”。也就是说，本文不只重复“有图谱、有 RAG、有 Docker”，而是强调这些能力如何被组合成可评测、可追溯、可部署、可审阅的系统。

## 1. 自建 RAG Benchmark 自动优化闭环

赛题鼓励选手自建 20-50 道 RAG Benchmark，并用数据驱动方式优化 pipeline。本项目将这个要求做成自动化闭环，而不是人工写一张静态表。

实现方式：

- 从已解析教材 chunk 中自动生成 30 道题。
- 题型覆盖 definition、fact、source、compare、reasoning、cross_textbook。
- 每题保存预期教材、章节、页码、chunk id、关键词和答案提示作为 ground truth。
- 自动遍历 chunk size、overlap、稀疏检索模型、phrase rerank，共 48 组配置。
- 用 Recall@5、MRR、答案准确率、引用命中率、证据命中率、平均响应时间和估算 token 成本评估。
- 将最优配置写入 `src/backend/app/rag_defaults.json`，线上 RAG 默认读取该配置。

数据结果：

- 最优配置：`chunk500_ov100_char_2_5_phrase`
- Recall@5：0.6667
- 答案准确率：0.5667
- 引用命中率：0.3667
- 证据命中率：0.5667
- 平均耗时：78.35ms
- 平均上下文 token：1176.07

创新价值：

- 将“RAG 调参”从经验判断变成可复验实验。
- 让架构文档中的 RAG 设计有量化证据。
- 符合 P1 与 P2 中“有实验数据支撑”的加分方向。

## 2. 中文教材混合检索与短语 rerank

医学/课程中文教材没有天然空格，普通词级 BM25 容易把核心术语拆散。例如“肾上腺素这个概念的核心定义是什么”中，“概念、核心、定义、是什么”会干扰真正的实体词。

本项目采用：

- TF-IDF 字符 ngram 召回。
- BM25 关键词召回。
- 中文 2-5 字 ngram 补充召回。
- 明确概念短语加权 rerank。
- 命中明确证据词时优先返回原文证据句和引用。

创新价值：

- 不依赖大型 embedding 模型，部署轻量。
- 对中文专业术语更稳。
- 与 benchmark 自动优化联动，而不是孤立规则。

## 3. 知识图谱质量可视化编码

知识图谱不是简单画点线，而是把抽取质量和教学重要性编码到视觉元素中。

编码规则：

- 节点大小：连接度、频次、定义深度、来源文本长度、质量分综合评分。
- 节点颜色：教材来源。
- 大深色节点：高频、高连接或多证据核心概念。
- 虚线描边：低质量或含抽取 warning 的节点。
- 标签显示：缩放到阈值后显示，避免小尺度文字拥挤。
- 关系样式：contains、prerequisite、applies_to、parallel 使用不同颜色/线型/箭头。

创新价值：

- 教师能快速区分核心概念、局部概念和可疑节点。
- 与后端 `quality_score/extraction_method/warnings` 形成可解释闭环。
- 优于只把知识点静态渲染成同尺寸节点的基础图谱。

## 4. 整合决策可解释性溯源

跨教材整合不是只给一个合并后的节点，而是保留可审计决策。

每条决策包含：

- `decision_id`
- `action`: merge / keep / remove
- `affected_nodes`
- `result_node`
- `reason`
- `confidence`
- `status`

真实报告中的当前统计：

- 原始教材数量：3
- 原始总字数：1,343,665
- 整合后字数：83,565
- 压缩比：6.22%
- 节点数：1217 -> 1161
- 关系数：671 -> 670
- 合并：53 项
- 保留：1108 项
- 删除：0 项

创新价值：

- 把压缩结果变成“可解释、可复核、可追溯”的教学整合方案。
- 报告案例直接展示概念名、来源教材、章节和页码。
- 教师反馈可以围绕具体决策提出修改意见。

## 5. 教师审阅一体化工作台

项目把教材管理、图谱、整合、RAG、对话、报告放在同一 SPA 中，减少教师在多个工具间切换。

体验细节：

- 上传教材有进度反馈。
- 构建图谱和整合有进度条，完成后隐藏。
- 右侧功能面板独立滚动，避免全局滚动破坏图谱视野。
- RAG 引用显示教材、章节、页码、相关度分数和原文 chunk。
- 教师反馈保留会话历史。

创新价值：

- 更贴近真实教研工作流。
- 把 AI 输出从一次性生成变成可审阅、可反馈、可追溯的工作台。

## 6. 单端口生产部署和 ModelScope 适配

赛题要求公网可访问链接。本项目避免前后端双端口带来的跨域和代理不确定性，生产模式由 FastAPI 同源托管 React 构建产物。

实现方式：

- `/` 返回 React 前端。
- `/assets/*` 返回前端静态资源。
- `/api/*` 返回后端接口。
- Docker 暴露单端口。
- `app.py` 读取 `PORT`，默认适配 ModelScope 的 `7860`。

创新价值：

- 降低比赛部署平台的不确定性。
- 同一份代码可用于本地 Docker、ModelScope 和临时公网隧道。
- 更容易通过评审的“打开浏览器能正常使用”验收。
