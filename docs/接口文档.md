# 接口文档

## 教材

### `POST /api/textbooks/upload`

`multipart/form-data`，字段 `files` 可传多个 PDF/MD/TXT/DOCX。

返回：

```json
{
  "textbooks": [
    {
      "textbook_id": "book_xxx",
      "filename": "生理学.pdf",
      "status": "completed",
      "chapter_count": 12,
      "total_chars": 123456
    }
  ]
}
```

### `GET /api/textbooks`

返回教材摘要列表。

### `GET /api/textbooks/{textbook_id}`

返回教材完整章节结构。

## 图谱

### `POST /api/graphs/build`

请求：

```json
{
  "textbook_ids": ["book_xxx"],
  "use_llm": true
}
```

`textbook_ids` 省略时构建全部已解析教材。

### `GET /api/graphs/{textbook_id}`

返回节点和边；`textbook_id=integrated` 时返回整合后图谱。

## 整合

### `POST /api/integration/run`

运行跨教材整合，返回决策、整合图谱和统计。

### `GET /api/integration/decisions`

读取当前整合状态。

### `POST /api/integration/feedback`

请求：

```json
{
  "message": "请保留炎症相关知识点"
}
```

返回更新后的整合状态。

## RAG

### `POST /api/rag/index`

对当前教材建立 chunk 索引。

### `GET /api/rag/status`

返回索引状态。

### `POST /api/rag/query`

请求：

```json
{
  "question": "炎症是什么？"
}
```

返回：

```json
{
  "answer": "……[病理学, 第四章 炎症, 第78页]",
  "citations": [
    {"textbook": "病理学", "chapter": "第四章 炎症", "page": 78, "relevance_score": 0.91}
  ],
  "source_chunks": ["原文 chunk"]
}
```

## 报告

### `GET /api/report/integration`

根据当前系统状态写入并返回 `report/整合报告.md`。
